# Backend Development Guidelines\n\nThis section covers the FastAPI backend implementation for the Multi-User Todo App.\n\n## Technology Stack\n\n- **Framework**: FastAPI\n- **Language**: Python\n- **ORM**: SQLModel\n- **Database**: Neon Serverless PostgreSQL\n- **Authentication**: Better Auth with JWT tokens\n\n## Key Principles\n\n- Follow RESTful API design principles\n- Implement proper authentication and authorization\n- Use Pydantic models for request/response validation\n- Handle errors gracefully with appropriate HTTP status codes\n- Implement proper logging\n\n## API Structure\n\n- Define API routes in dedicated router modules\n- Use dependency injection for authentication checks\n- Implement proper database session management\n- Follow consistent response formats\n\n## Database Operations\n\n- Use SQLModel for defining database models\n- Implement proper relationships between entities\n- Use Alembic for database migrations\n- Follow proper transaction management\n\n## Authentication & Authorization\n\n- Integrate Better Auth for user management\n- Validate JWT tokens in protected endpoints\n- Implement user isolation for multi-user functionality\n- Secure sensitive endpoints\n\n## Best Practices\n\n- Use environment variables for configuration\n- Implement proper rate limiting\n- Document APIs with OpenAPI/Swagger\n- Write unit tests for business logic
